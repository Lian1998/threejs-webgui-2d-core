# 版本1: 基础实现逻辑
```
1. 假设原本顶点是
a(x1, y1, z1), b(x2, y2, z2), c(x3, y3, z3)

2. 先将顶点扩充成两份
positions: x1, y1, z1, x1, y1, z1, x2, y2, z2, x2, y2, z2, x3, y3, z3, x3, y3, z3

3. 在vertexShader会计算当前顶点的法线
4. 将当前顶点扩充出来的两个顶点, 沿着法线和法线的负方向扩充半个单位的宽度
```

# 版本2: 保证线宽有两种画法
```
1. 按世界位置算(线宽随着缩放而缩放)
2. 按屏幕空间位置(线宽依据于像素固定值不会随着缩放而缩放)
```

# 版本3: (一次position)合并drawcall
```
1. 每条线段的起点和终点都重复一遍
2. 添加一个attribute用来记录此顶点是否为线条的起点和终点

第一条线(三个顶点):
1 (0 0 0) 1
1 (0 0 0) 1

第二条线(两个顶点):
1 (0 0) 1
1 (0 0) 1

3. 在shader中保证在 1, 1 线段在fragmentShader中被抛弃
```
